# Mode Formule 1 — WC2026 Pool

Documentation du mode F1 (MVP) : API, limites, règles de paris et architecture.

## Vue d'ensemble

Le mode F1 coexiste avec le football via un **switch global** (header) persisté dans `profiles.active_sport` + cookie `active_sport`. Les routes F1 sont sous `/f1/...` pour éviter toute régression sur le mode CDM 2026.

## API utilisée

### OpenF1 (prioritaire)

- **Base URL** : `https://api.openf1.org/v1`
- **Auth** : aucune pour les données historiques (2023+)
- **Live** : données temps réel ~3 s de décalage ; tier gratuit avec rate limits (~3 req/s, 30 req/min)
- **Endpoints MVP** :
  - `GET /meetings?year=2026` — calendrier GP
  - `GET /sessions?meeting_key={key}` — sessions (Practice, Qualifying, Race)
  - `GET /drivers?meeting_key={key}` — line-up pilotes
  - `GET /session_result?session_key={key}` — résultats officiels (règlement auto)
  - `GET /position?session_key={key}` — positions live
  - `GET /race_control?session_key={key}` — drapeaux, SC, messages
  - `GET /weather?session_key={key}` — météo piste

### Fallback / complément

- **Jolpica F1** (ex-Ergast) : `https://api.jolpi.ca/ergast/f1/current/`
  - Classement pilotes / constructeurs (`/f1/standings`) — cache Next.js 1 h
  - Calendrier / standings si OpenF1 indisponible (fallback futur)

## Paris MVP

### Vainqueur GP (`race_winner`) — legacy

Remplacé par le top 10 + bonus P1. Conservé en base pour les paris déjà placés.

### Top 10 ordonné (`race_order`)
- **Clôture** : départ course (`race_start_at`)
- **Barème** (par pilote prédit) :

| Écart vs position réelle | Points |
|--------------------------|--------|
| Exact | 10 |
| ±1 place | 6 |
| ±2 places | 3 |
| ±3 places | 1 |
| > 3 | 0 |

- **Max** : 115 pts (100 positions + **15 bonus** si vainqueur exact en P1) · Boost ×2 · On Fire au règlement
- **Règlement** : `f1_meetings.race_results` (OpenF1 `session_result`)
- RPC : `place_f1_race_order_bet(meeting_key, order jsonb, use_boost)`
- **UI** : choisir 10 pilotes dans la grille, puis glisser-déposer pour ordonner

### Pole position (`pole_position`)

- **Clôture** : début des qualifications (`quali_start_at`)
- **Points** : cote fixe 6 → ~17 pts (`points_from_odd`)
- **Règlement** : `f1_meetings.pole_driver_number` (OpenF1 quali `session_result`)
- RPC : `place_f1_pole_bet`

### Duel coéquipiers (`teammate_duel`)

- **Clôture** : départ course
- **Points** : cote fixe 1.9 → ~11 pts
- **Règlement** : position finale course vs coéquipier (même `team_name`)
- RPC : `place_f1_teammate_duel_bet`
- Un seul duel par week-end (choix parmi les paires d'écuries)

### Engagement week-end

- **Pass week-end** : progression pole + duel + top 10
- **Timeline** : sessions FP / Quali / Sprint / Race (OpenF1)
- **Debrief** : tableau delta position par position après course
- **Partage** : bouton « Partager ma grille » (Web Share API ou presse-papier)

### Hors scope

- Meilleur tour, premier abandon, head-to-head multi-GP

## Live sur circuit

Page `/f1/[meetingKey]` en statut `live` :

- Bandeau drapeau / météo (OpenF1 race_control + weather)
- Image circuit (`circuit_image` OpenF1)
- Classement provisoire (polling `/api/f1/live` toutes les 4 s)

## Classement

- Route dédiée `/f1/leaderboard` — stats pool (paris gagnés / perdus / points F1)
- Route `/f1/standings` — championnat mondial pilotes & constructeurs (Jolpica)
- **Séparé** du classement football (`/leaderboard`)

## Admin

- `/admin/f1` : sync manuelle, forcer statut live, corriger vainqueur + régler paris
- Toggle `f1_mode_enabled` pour désactiver le mode côté joueurs

## Limites connues

1. **Cotes** : pas de bookmaker F1 gratuit ; cotes estimées en MVP
2. **Live OpenF1** : tier gratuit = données historiques sans auth ; live peut nécessiter abonnement OpenF1 selon période (doc OpenF1 : live = fenêtre session ±30 min)
3. **Rate limits** : sync séquentielle avec throttle 350 ms entre requêtes
4. **Points globaux** : les gains F1 créditent le même solde `profiles.points` (ligues/collection restent mode football)
